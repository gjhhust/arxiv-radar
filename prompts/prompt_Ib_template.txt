**论文信息**
- arxiv ID：{arxiv_id}
- 标题：{paper_title}

---

## Step 1：获取论文内容

使用 arxiv-fetch 技能获取论文全文：

```
arxiv-fetch {arxiv_id}
```

将返回的摘要、正文（或 LaTeX 源码）保留在上下文中，后续分析使用。

---

## Step 2：获取 S2 引用列表（立刻执行，在分析前完成）

运行以下命令，获取本论文在知识图谱中的完整引用列表：

```
sqlite3 /Users/lanlanlan/.openclaw/workspace/skills/arxiv-radar/data/paper_network.db "SELECT p.title, p.id FROM paper_edges e JOIN papers p ON e.dst_id = p.id WHERE e.src_id = '{arxiv_id}' AND e.edge_type = 'CITES' ORDER BY p.title"
```

将查询结果（title 列表 + arxiv_id）保存为"S2 引用列表"，供 Step 3 的 core_cite 使用。

**⚠️ core_cite 每条 title 必须来自此 S2 引用列表，arxiv_id 直接取查询返回的 p.id，不联网验证。**

---

## Step 3：分析论文，输出 JSON

基于 Step 1 获得的论文内容和 Step 2 获得的 S2 引用列表，完成以下字段：

cn_oneliner
  一句话，严格格式：「基于[前驱方法/范式]，引入[核心改动]，实现[效果/指标]」
  例1：「基于 Transformer 自注意力，引入局部窗口限制将复杂度从 O(n²) 降至 O(n)，实现长序列高效建模」
  例2：「基于 DDPM 扩散过程，将噪声预测迁移至 VAE 潜空间，以 8× 压缩比实现高分辨率图像生成」

cn_abstract
  2-4 句完整中文摘要，保留核心数值结果，不截断

contribution_type
  严格四选一：incremental / significant / story-heavy / foundational

editorial_note
  三段连续文字，80-150 字：
  [段1：前驱方法的核心贡献与局限] → [段2：本文如何针对性解决] → [段3：编辑判断：创新程度、适用场景、潜在不足]

why_read
  一句话：[目标读者群体] + [读后具体收获]

method_variants
  JSON 数组，记录本文对已有具体方法的实质性改造。每项包含：
  - base_method：已有具体方法名（小写英文，如 vqgan、titok、nested-dropout、flextok）
  - variant_tag：格式 "base_method:改动标签"
  - description：一句话，格式「[base_method] 原本做 X；本文将其改为 Y，以解决 Z」

  例：[
    {"base_method": "nested-dropout",
     "variant_tag": "nested-dropout:redundant-padding",
     "description": "nested-dropout 随机丢弃尾部 token 实现弹性长度；本文改为重复填充尾部 token 提高激活频率，解决尾部 token 训练不足导致的质量下降"},
    {"base_method": "vqgan",
     "variant_tag": "vqgan:semantic-reg",
     "description": "VQGAN 用纯重建目标优化量化空间；本文加入 DINOv2 对齐损失约束 latent 复杂度，使 AR 模型更易学习 token 分布"}
  ]
  ⚠️ 若无实质性改造，返回空数组 []
  ⚠️ 不要把论文自身的不同规模版本列为 variants

core_cite
  ≥10 条，每条 title 必须来自 Step 2 的 S2 引用列表（输出前逐条核查），包含：
  - title：来自 S2 引用列表的论文标题
  - arxiv_id：来自 Step 2 查询返回的 p.id（直接使用，不联网验证）
  - role：五选一 extends | contrasts | uses | supports | mentions
  - note：一句话说明引用原因

  role 定义：
  extends   — 本文直接在此工作基础上扩展
  contrasts — 本文与此工作对比或批判
  uses      — 本文直接使用其方法/代码/数据集
  supports  — 作为背景支撑，引用以说明动机
  mentions  — 简单提及，非核心

idea
  3 条研究方向，必须分别来自以下三类来源（每类至少一条）：
  A. 本文局限/作者明确说没做的方向（来自 Limitation 或 Future Work 章节）
  B. 引用列表中"被本文用到但能力未充分挖掘"的方法——其原本还能做什么？
  C. 邻域借鉴——相邻子领域中有无解决类似问题的方法可迁移？大跨（NLP→视觉生成、音频→图像 tokenization）和小跨（视频生成→表征学习、3D 重建→视频生成、image AR→video AR）都算，不必强求跨出 CV。

  每条包含：
  - title：研究方向标题（10-20 字）
  - why：2-3 句。第一句说清楚 gap（本文为什么没做到），后续说可行性依据。

  ⚠️ 禁止：把本文已经做到的事情列为 idea
  ⚠️ 避免：纯粹"换 backbone"、"加更多数据"、"扩展到更多任务"等拼凑型 idea

  ],
  "idea": [
    {"title": "...", "why": "..."}
  ]
}
