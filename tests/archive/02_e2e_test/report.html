cn_abstract 现有图像tokenizer（如VQGAN）将图像压缩为2D网格latent，固定的空间对应关系限制了对图像冗余的利用。本文提出TiTok（Transformer-based 1-Dimensional Tokenizer），将图像patchify后与可学习的1D latent tokens拼接输入ViT encoder，仅保留latent tokens作为图像表示，再通过ViT decoder结合mask tokens重建图像，彻底解耦latent尺寸与图像分辨率。为绕开复杂GAN训练配方，提出两阶段训练：先用现成MaskGIT-VQGAN生成proxy codes进行warm-up，再fine-tune decoder至像素空间。TiTok-L-32仅用32个token即在ImageNet 256×256达到rFID 2.21，配合MaskGIT生成框架取得gFID 1.97；在512×512上以410倍速度超越DiT-XL/2（gFID 2.74 vs 3.04）。 TiTok是一种基于Transformer的1D图像分词器，将传统2D latent grid转换为1D序列表示。通过将256x256x3图像压缩至仅32个离散tokens，相比传统2D方法实现8-64倍压缩率。采用ViT encoder-decoder架构结合向量量化，并通过两阶段训练策略（proxy codes预热+decoder fine-tuning）提升性能。在ImageNet 256x256上达到1.97 gFID，比MaskGIT提升4.21；在512x512上超越DiT-XL/2且生成速度快410倍。 本文提出Transformer-based 1-Dimensional Tokenizer（TiTok），将传统VQGAN/VQ-VAE的2D latent grid改为可变长度的1D latent sequence：在ViT encoder中把patch tokens与K个learnable latent tokens拼接，仅保留latent tokens并做vector quantization，再在ViT decoder端将quantized latents与复制的mask tokens拼接以重建图像。为避免VQGAN训练配方（perceptual+GAN loss等）难以复现带来的不稳定，作者提出两阶段训练：先用开源MaskGIT-VQGAN产生的proxy codes监督训练TiTok（输出proxy code序列并接原VQGAN decoder出RGB），再冻结encoder/quantizer对decoder做像素域fine-tuning。作为MaskGIT的替换tokenizer，TiTok在ImageNet 256/512上以32~128 tokens获得与SOTA相当或更优的rFID/gFID，并因token数大幅减少带来显著训练/采样加速（最高报告410×）。

cn_abstract 自回归图像生成中，视觉tokenizer规模增大会改善重建质量但降低下游AR生成质量，本文称之为重建vs生成困境。GigaTok通过语义正则化损失将tokenizer decoder中间特征与冻结DINOv2特征对齐，防止潜空间复杂度随规模增长而过度膨胀。此外，作者提出三项scaling实践：使用基于Q-Former的1D tokenizer、优先扩展decoder、以及用entropy loss稳定十亿级tokenizer训练。最终GigaTok扩展至3B参数，在ImageNet上的重建、AR生成和AR表示质量上均达到SOTA。 本文研究自回归图像生成中视觉tokenizer的扩展难题。针对扩展tokenizer导致重构质量提升但生成质量下降的矛盾，作者提出语义正则化方法，将tokenizer特征与预训练DINOv2特征对齐以约束潜在空间复杂度。结合1D tokenizer架构、非对称编解码器扩展和entropy loss三大实践，成功训练3B参数tokenizer，在ImageNet上同时实现最优重构、重构与生成性能。 本文研究AR image generation两阶段框架中visual tokenizer的扩容问题，发现tokenizer参数变大虽提升重建(rFID/LPIPS)，却会让下游next-token prediction的AR模型更难学习token分布，导致生成(gFID)变差。作者提出semantic regularization：用冻结的DINOv2特征监督，对齐tokenizer decoder中间层特征与语义表征，以抑制latent space complexity增长，从而缓解“reconstruction vs generation dilemma”。在此基础上，论文给出可扩展实现：采用Q-Former构建1D tokenizer、优先扩大decoder的asymmetric scaling、并用entropy loss稳定billion-scale VQ训练，最终将tokenizer扩到3B并在ImageNet 256上同时提升重建、生成与AR表示线性探测精度。

cn_abstract 现有灵活图像tokenizer采用nested dropout训练，导致图像信息过度集中于早期token，使得下游AR生成质量随token数增加无法持续提升。本文提出ReTok，引入Redundant Token Padding（在序列尾部拼接冗余query token，迫使原始尾部token更频繁被激活）和Hierarchical Semantic Regularization（对早期token施加较强的DINOv2语义对齐约束，并向尾部token线性衰减以保留细节重建能力）。在ImageNet 256×256上，ReTok在所有灵活tokenizer中取得最优生成性能（gFID 2.27），并与固定长度tokenizer持平。 灵活图像tokenizer使用嵌套dropout实现可变长度序列，但信息过度集中于早期token，限制了下游自回归生成质量的提升。本文提出ReTok，通过redundant token padding增加尾token激活频率，缓解信息过度集中；同时引入hierarchical semantic regularization，将早期token的解码特征与DINOv2预训练视觉模型对齐，并随序列长度递减正则化强度，实现从粗粒度语义到细粒度细节的渐进重建。在ImageNet 256×256上验证，ReTok实现了优于现有灵活tokenizer的生成性能。 本文分析 flexible image tokenizers 采用 nested dropout 的“尾部截断”训练会把信息过度压缩到前缀 token，导致下游 autoregressive (AR) 生成在 token 变长时 gFID 不升反降且显著落后 fixed-length tokenizer。为此提出 ReTok：在训练时用 Redundant Token Padding 在序列尾部追加额外 query tokens，使原序列的尾 token 在 dropout 下更频繁被激活，从而缓解信息前置；同时用 Hierarchical Semantic Regularization 将短前缀解码特征对齐到预训练 DINOv2 语义特征，并随保留长度 k 线性衰减正则强度以保留后缀细节重建能力。实验在 ImageNet 256×256 上表明 ReTok 使 AR 生成质量随 token 长度更一致地提升，并在同等 token 预算下优于现有 flexible tokenizers，接近/达到强 fixed-length 1D tokenizer 的水平。

editorial_note [前驱] 本文直接建立在GigaTok（1D固定长度tokenizer+语义正则化）和nested dropout（flexible tokenizer训练范式）之上，AR生成器使用LlamaGen，语义对齐来自DINOv2；FlexTok、One-D-Piece等flexible tokenizer是直接对比基线。 [贡献] 作者发现naive nested dropout使信息过度集中于早期token，提出两个训练技巧：（1）填充冗余token让原始尾部token变成早期token从而被更多激活；（2）对不同长度的truncated序列施加渐变强度的语义正则化。 [判断] 贡献清晰有效，但属于对已知问题的工程修补：padding trick本质是数据增强的变体，hierarchical regularization是已有表示对齐的平滑扩展。解决了实际痛点，实验充分，但缺乏深层理论洞见，属于有效但可预期的改进。 [前驱]本文建立在GigaTok的1D tokenizer架构之上，采用了FlexTok/One-D-Piece的nested dropout训练策略，并继承了GigaTok中用DINOv2做representation alignment的思路。核心模块分别来自：tokenizer架构(GigaTok)、灵活长度训练(nested dropout from FlexTok)、语义正则化(DINOv2)。[贡献]作者真正做的事情是：提出redundant token padding（通过追加padding tokens让原始序列的尾token在nested dropout中被更多激活）和hierarchical semantic regularization（对不同位置的token施加递减的语义对齐约束），从而解决灵活tokenizer在AR生成中「token越多质量却不提升」的已知瓶颈。[判断]这个贡献有实质价值：作者清晰识别并解决了领域内公认的问题（FlexTok、One-D-Piece等都有这个瓶颈），提出的解法具体有效，不是简单的工程堆砌，在灵活tokenizer这个子方向上有推进作用。 [前驱] 工作建立在 1D 离散 tokenizer（TiTok/GigaTok）与 flexible tokenizer 的 nested dropout（FlexTok/One-D-Piece/DetailFlow）之上，并沿用 tokenizer 训练中的 representation alignment/semantic regularization（REPA、GigaTok 对齐 DINOv2）。 [贡献] 核心新意是两条训练技巧：训练时在序列尾部加“冗余 padding tokens”改变 nested dropout 的激活分布；再把语义对齐权重做成随保留长度递减的分层正则，并额外做 decoder 微调。 [判断] 这是针对已知症状（尾 token 不工作）的一组直接修补，效果明确但方法学深度有限；更像把训练分布与正则日程调到更匹配 AR 需求。

nested-dropout:redundant-token-padding 原始nested dropout直接随机截断尾部token；本文在原始N个token后拼接M个冗余query token构成N+M序列，再做nested dropout，使原始尾部token的激活频率提升，训练完后丢弃冗余token只用原始N个token做下游生成。
gigatok:hierarchical-semantic-regularization GigaTok对所有长度的decoder中间特征施加固定权重λ的DINOv2余弦相似度正则；本文将λ设为关于保留序列长度k的线性递减函数，短序列施加强语义约束，长序列衰减至低权重以支持细节重建。
gigatok:decoder-finetuning 训练完编码器和量化器后，冻结二者并以小常数λ微调decoder，缓解强语义约束对短token序列重建质量的负面影响。
gigatok:redundant-token-padding 原gigatok采用固定长度token序列；本文在训练时追加padding tokens到序列尾部，使nested dropout能激活原始序列的尾token，缓解信息过度集中于早期token的问题
gigatok:hierarchical-semantic-reg 原gigatok对所有token使用恒定的语义正则化强度；本文改为对不同长度k的保留序列使用递减的正则化权重λ(k)，早期token强调语义对齐，尾token允许细节重建
flextok:decoder-finetune 原flextok直接用训练好的decoder；本文在训练后对decoder进行微调，固定encoder和quantizer，降低语义约束以提升短token序列的重建质量
nested-dropout:redundant-padding 原方法随机截断尾 token 迫使前缀重建，易导致信息前置；本文在训练时追加 M 个 padding query/token 后再做截断，使原序列尾 token 更常被保留与训练，推高尾 token 利用率。
gigatok:flexible-nested-dropout-ft 原 GigaTok 是 fixed-length 1D tokenizer；本文先用 naive nested dropout 将其微调成 flexible 版本以暴露 AR 生成瓶颈，并作为对照。
representation-alignment:hierarchical-lambda(k) 原对齐用固定权重将 tokenizer decoder 中间特征与 DINOv2 特征做 cosine 对齐；本文令正则权重 λ 随保留长度 k 线性衰减，短序列更偏语义、长序列更偏细节。
vit-decoder:finetune-low-reg 原训练同时优化 encoder/quantizer/decoder 且短序列受强语义约束可能伤重建；本文冻结 encoder+quantizer，仅以较小常数 λ 微调 decoder，提升短 token 重建与短序列生成。
