现有flexible image tokenizer通过nested dropout训练，导致图像信息过度集中于早期token，使得下游AR模型生成质量随序列长度增加无法持续提升，甚至退化。本文提出ReTok，引入Redundant Token Padding（在序列尾部附加冗余query token后再执行nested dropout，迫使原始序列尾部token得到更多激活）和Hierarchical Semantic Regularization（对早期token施加强DINOv2语义对齐约束，随序列长度增加线性衰减以允许尾部token学习细节重建），并配合decoder fine-tuning进一步改善短序列生成质量。在ImageNet 256×256上，ReTok-S-B+LlamaGen-XL达到gFID 2.27，在flexible tokenizer中达到最优，并与同规模固定长度tokenizer持平。 本文针对基于nested dropout的flexible image tokenizer在autoregressive（AR）图像生成中存在的质量瓶颈问题展开研究：随着生成token数量增加，生成质量无法持续提升，且显著落后于fixed-length tokenizer。为此提出ReTok，引入Redundant Token Padding策略，在原始序列尾部拼接冗余query tokens，通过nested dropout使尾部tokens获得更高激活频率，从而缓解信息过度集中于早期tokens的问题。进一步提出Hierarchical Semantic Regularization，对早期tokens施加强语义对齐约束（对齐DINOv2特征），并随序列长度增加线性衰减约束强度，使尾部tokens专注于细节重建。在ImageNet 256×256上，ReTok在flexible tokenizer中取得最优生成性能，并接近同等规模fixed-length tokenizer水平。 现有 flexible image tokenizers 通过 nested dropout 随机截断尾部token训练，但会把信息挤压到前缀token，导致下游 autoregressive (AR) 生成在token变长时质量不升反降，且同token预算下显著落后 fixed-length tokenizer。本文提出 ReTok：在训练时对序列尾部追加 redundant token padding 提高原始尾token被保留/参与重建的频率，并用 hierarchical semantic regularization 将短序列(早token)的decoder特征对齐到 DINOv2 语义特征、随保留长度k增大线性衰减约束以释放尾token重建细节，最后再做decoder fine-tuning。ImageNet 256×256 上，ReTok 使 gFID 随token长度增长更稳定提升，在灵活tokenizer中优于 One-D-Piece/DetailFlow，并在256 tokens时达到接近 GigaTok 固定长度基线的生成质量（如 ReTok-S-B+LlamaGen-L gFID 2.66，+LlamaGen-XL 2.27）。 本文分析 flexible image tokenizers 采用 nested dropout 训练时的"尾部截断"会迫使信息过度集中在前缀 token，导致下游 autoregressive (AR) 图像生成在 token 变长时质量不升反降、且显著落后于 fixed-length tokenizer。作者提出 ReToK：在保持 GigaTok 式 1D ViT+quantizer 架构不变的前提下，通过 Redundant Token Padding 在训练时把序列尾部扩展到更长长度以提高原序列尾部 token 的被保留频率，并用 Hierarchical Semantic Regularization 将 decoder 中间特征与 DINOv2 语义特征对齐且随保留长度 k 线性衰减正则强度以兼顾语义与细节。实验在 ImageNet 256×256 上表明，ReToK 相比现有 flexible tokenizers（如 One-D-Piece、DetailFlow）在多种 token 长度下更稳定提升 gFID，并在 256 tokens 下达到接近强 fixed-length 1D tokenizer（GigaTok）的生成质量。 灵活图像tokenizer使用nested dropout训练时，信息过度集中于早期tokens，导致生成更多tokens时质量不升反降。本文提出ReTok，通过冗余token填充在序列尾部添加额外tokens以激活尾部tokens，并采用层级语义正则化让早期tokens学习语义特征、尾部tokens专注细节重建。实验表明，在ImageNet 256×256上，ReTok-S-B配合LlamaGen-XL达到gFID 2.27，显著优于其他灵活tokenizer。 灵活图像tokenizer使用nested dropout随机截断尾部token进行训练，导致图像信息集中在早期token，限制了下游自回归生成质量。本文提出ReTok，通过冗余token填充增加尾部token激活频率，并采用分层语义正则化将早期token解码特征与DINOv2对齐，逐步降低对后续token的正则化强度以允许重建更精细的图像细节。实验表明，该方法在ImageNet 256×256上取得优于现有灵活和固定长度tokenizer的生成性能。

本文核心贡献是识别并量化了flexible tokenizer（nested dropout训练）在下游AR生成中的'尾部信息集中'问题，并提出两个补丁：冗余padding是一个训练技巧（通过扩展序列长度让原始尾部token变成'早期token'），层次语义正则化则直接沿用GigaTok的representation alignment思路加了一个线性decay权重。两个组件均为已有方法（GigaTok的语义正则化 + nested dropout的变体）的直接组合，方法上几乎没有新原语；真正有价值的是对flexible tokenizer AR生成瓶颈的系统性诊断，但这一发现的新颖性也有限，FlexTok论文中已有类似现象的观察。 **[前驱关系]** 本文直接建立在GigaTok（fixed-length 1D tokenizer + representation alignment框架）和nested dropout（来自FlexTok/One-D-Piece）的基础上；tokenizer架构完全继承GigaTok-S-S/S-B，语义正则化直接沿用GigaTok的DINOv2对齐损失设计，LlamaGen作为AR生成器评估平台。**[核心贡献]** 作者诊断了flexible tokenizer的生成瓶颈根因（尾部tokens训练曝光不足），提出两个修复：①在序列后拼接冗余tokens使原始尾部tokens变为中间tokens从而提升曝光频率；②对语义正则化权重随序列长度线性衰减。**[批判判断]** 问题诊断清晰且真实，但两个解决方案均属显而易见的工程补丁——padding改变tokens位置分布、正则化权重调度均无方法创新，属于在已有框架上打补丁、诊断价值高于方法价值的典型incremental工作。 核心贡献其实是对"nested dropout 导致信息前缀拥塞、尾token失活"这一训练分布偏置给出两个很工程化的补丁：用padding改变k的覆盖分布、用随k衰减的DINO对齐稳住短序列语义。有效但不算新范式：方法几乎完全建立在 GigaTok 架构与 representation alignment 既有套路上，创新点主要在于把这些组件按"激活尾token/分层语义-细节分工"的诊断重新组合并做了较完整的下游AR验证。 [前驱关系] 架构基本沿用 GigaTok/TiTok 的 1D query-token ViT tokenizer + VQ 训练，并继承 FlexTok/One-D-Piece 的 nested dropout 以实现可变长度；语义约束直接借鉴 REPA/GigaTok/VA-VAE 的 DINOv2 表征对齐思路。 [核心贡献] 作者指出"尾部截断"导致尾 token 训练曝光不足，从而提出在训练时额外拼接 padding tokens 来把原尾部变成"中段"，并把语义对齐权重设计成随 k 递减的 schedule。 [批判判断] 诊断问题较到位且对 AR 评测更聚焦，但方法本质是训练采样分布与正则权重的工程修补；效果依赖既有强 tokenizer 初始化与 DINOv2，对更广设置的泛化与理论解释仍偏弱。 本文核心贡献是发现了flexible tokenizer的尾部token利用不足问题，并提出两个训练技巧（冗余padding + 层级正则化）来缓解。这是典型的「发现问题+组合已有方法打补丁」工作：方法上完全基于GigaTok架构 + representation alignment思想，唯一的增量创新是将语义正则化从统一权重改为层级衰减。实验结果显示其达到了与固定长度tokenizer相当的性能，但未证明比直接使用固定长度tokenizer有任何架构优势。 **[前驱]** 方法直接基于GigaTok的1D tokenizer架构，继承了其encoder-decoder设计和codebook量化，并采用FlexTok提出的nested dropout训练策略，同时引入DINOv2的representation alignment。**[贡献]** 作者发现尾部token因nested dropout截断频率低而信息稀疏，提出两个修复：1）在训练时padding冗余token使尾部token被更频繁激活；2）对早期token施加更强的语义正则化约束。**[判断]** 问题诊断准确（尾部token信息不足确实限制了AR生成），但解决方案是显而易见的工程补丁——padding和分层正则化都是直接可想到的思路，没有方法论创新，属于有效但不深刻的增量改进。
