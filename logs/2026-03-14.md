## 11:35 — 🚨 严重错误：Mox spawn prompt 精简导致 H1v2/H2v2 结构性合规失败

### 错误描述

Mox 在 spawn 子 agent 时，为节省 token **手动精简了 task prompt**，仅保留字段名和简短描述，
**删除了以下关键约束**：

1. `contribution_type` 四选一具体选项列表（incremental/significant/story-heavy/foundational）
2. JSON 输出格式示例（含所有字段的 key 名和结构）
3. `title` 字段的明确要求
4. `idea` 的 list[dict] 结构示例

### 影响范围

- **H1v2**：10/11 篇受影响（仅 2406.07550 未被精简，但 H1v2 整体也用了精简 prompt）
- **H2v2**：10/11 篇受影响（2406.07550 用了完整 task 内容，其余 10 篇精简）
- 对照：原版 H/Ia（使用完整 task 文件）contribution_type **100% 合规**

### 合规失败统计

| 检查项 | H1 | H2 | 说明 |
|--------|----|----|------|
| contribution_type 非标准 | 8/11 | 10/11 | 模型自创标签（`方法创新`、`Method`、`new_method` 等）|
| title 缺失 | 0/11 | 7/11 | H2 精简 prompt 未给 JSON 示例 |
| cn_oneliner 超长 | 11/11 | 11/11 | 模型普遍不遵守字数约束（两方案均有） |
| idea ≠ 3 | 0/11 | 5/11 | H2 返回 dict{A,B,C} 或 4 条 |
| core_cite < 10 | 0/11 | 1/11 | 2503.08685 只有 5 条 |
| core_cite 格式错 | 0/11 | 1/11 | 2506.05289 返回 list[str] 而非 list[dict] |

### 根因

Mox 的操作错误。task 文件内容完整无误，但 spawn 时自行精简 prompt 丢失了关键约束。

### 教训

> **spawn 子 agent 时必须使用完整 task 文件内容，禁止手动精简 prompt。**
> 任何精简都可能删除模型遵从格式约束所依赖的关键信息（选项列表、JSON 示例、字段类型等）。

- **contribution_type / title / idea 格式**：不可信，需重跑
- **core_cite 内容**：H1 全部合规；H2 有 2 篇结构异常

### 决策

H1v2 和 H2v2 的 **compliance 数据无效**，不能用于模板选择决策。
需要使用完整 task 文件重跑，或基于 editorial_note 等不受影响字段做内容质量对比。
